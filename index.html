Style Guide

Guidelines

Adopt-a-Tree uses Sass as a preprocessor for this project's CSS.

Watch and compile the Sass files using the command line, or a GUI application like Prepros, or a task runner like Gulp, or a bundler like Webpack.

This project uses node-sass via the command line using the following command:

node-sass --watch assets/scss/styles.scss assets/css/styles.css

Do use single-line Sass comments (which are the same as single-line JavaScript comments) for comments that other developers need to see, since they won't get compiled into the CSS stylesheet.

Don't edit the generated CSS files.

Don't use inline styles. The handful of inline styles in the style guide are purely for demo.

Do use 2 spaces to indent your code.

Do follow hyphen-separated class naming. E.g. list-block.

Modifier classes use a double-hyphen naming convention. E.g.list-block--profile.

Do create a separate class or ID for an element which requires JavaScript to modify its behavior. Prefix it with js- and don't attach any styling to this class or ID.

Do browse through the variables, globals, spacing and typography partials. They contain plenty of reusable code.

Do use camelCase for naming variables and functions in JavaScript.

Do use semicolons after JavaScript statements.

Do strive for an accessible user interface for users of keyboards and screen readers.

Do ensure the project works on modern browers, including IE Edge, Chrome, Firefox, Safari, Android and iOS.

Media Queries

The site primarily uses min-width media query variables since the layout is mobile-first. However, max-width media query variables are also available.

Min-width media query variables: $media-sm, $media-mid, $media-lg.

Max-width media query variables: $media-sm-max, $media-mid-max, $media-lg-max.

Media queries are usually nested within the selector they are associated with.

Reference files: _variables.scss

Click the "Code" button to see a contrived example of all media queries on the site.

Grid

To create a grid structure, start with a container, include a row and add columns. Use flex helpers with rows or float helpers with columns.

A 12 column grid is provided with classes for each media query breakpoint. E.g. col-12, col-6-sm, col-4-mid and col-3-lg.

To change the number of columns, change the value of the grid-column-count variable in the variables partial. Doing so will automatically create new columns across all breakpoints. For example, you could change the numbers of columns from 6 to 12 or 24.

Reference files: _grid.scss, _variables.scss

Grid: Containers

Container sizes are obtained from the media query variables.

Each container is horizontally centered and has left and right padding.

A full-width container also exists for sections that span the entire width of the screen.

Icons

Adopt-a-Tree's font icon set was created with the Icomoon app. It includes font icon families like Feather, Font Awesome, Icomoon and Typicons.

Each icon has a class with a prefix of at-icon-.

To update the icon set, click the 'Import Icons' button on the Icomoon website and upload the selection.json file from the font-icon folder inside the assets folder. Add or delete icons as needed. Download the new icon set, copy the contents of the CSS file into the font-icons partial and update the paths in the @font-face rule in the file.
